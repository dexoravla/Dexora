# 数据一致性验证脚本使用说明

## 脚本功能

`validate_data_consistency.py` 脚本用于验证每个action文件夹下4个相机文件夹的图片数量和bson文件的数据帧数是否一致。

## 验证内容

1. **相机文件夹图片数量一致性**：
   - 检查 `camera0_head`、`camera1_left_wrist`、`camera2_right_wrist`、`camera3_third_view` 四个相机文件夹
   - 验证每个相机文件夹中的图片数量是否相同
   - 支持 jpg、jpeg、png、bmp、tiff 格式的图片

2. **BSON文件数据帧数一致性**：
   - 检查 `episode_0.bson` 和 `xhand_control_data.bson` 文件
   - 验证bson文件内部的数据帧数是否与图片数量一致
   - `episode_0.bson`: 检查data字段中各个数组的长度
   - `xhand_control_data.bson`: 检查frames字段的长度

## 使用方法

### 1. 验证所有action文件夹
```bash
python3 validate_data_consistency.py
```

### 2. 验证指定的action文件夹
```bash
python3 validate_data_consistency.py --action action8
```

### 3. 显示详细输出
```bash
python3 validate_data_consistency.py --action action8 --verbose
```

### 4. 指定基础路径
```bash
python3 validate_data_consistency.py --path /path/to/your/data
```

## 命令行参数

- `--path, -p`: 要验证的基础路径（默认：当前目录）
- `--action, -a`: 只验证指定的action文件夹
- `--verbose, -v`: 显示详细输出
- `--help, -h`: 显示帮助信息

## 日志文件

脚本会自动将验证结果保存到 `logs` 目录下的日志文件中：

- 验证所有action: `logs/data_validation_YYYYMMDD_HHMMSS.log`
- 验证单个action: `logs/action_ACTIONNAME_validation_YYYYMMDD_HHMMSS.log`

日志文件包含每个action和episode的详细数据统计信息。

## 依赖要求

脚本需要以下Python包：
- `bson`: 用于读取bson文件
- `pathlib`: 用于路径操作
- `glob`: 用于文件匹配

安装依赖：
```bash
pip install pymongo  # 包含bson模块
```

## 注意事项

1. 脚本会自动跳过不存在的相机文件夹或bson文件
2. 对于无法读取的bson文件，会显示警告信息但不中断验证过程
3. 验证过程可能需要较长时间，特别是当action文件夹数量很多时
4. 建议先使用 `--action` 参数测试单个action文件夹，确认脚本工作正常后再验证全部
